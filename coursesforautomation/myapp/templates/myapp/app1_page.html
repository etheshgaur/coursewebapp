What is Selenium?

Selenium stands as a highly regarded open-source software testing framework, primarily developed for automating web browsers (Chrome, Edge, Firefox, etc.).

This tool empowers both testers and developers to craft scripts using a variety of programming languages, such as Java, Python, and C#. These scripts enable interaction with web applications, automated testing procedures, and the validation of web-based features.

  • It allows testers and developers to write scripts in various programming languages (such as Java, Python, and C#) to interact with web applications.
  • Perform automated testing and validate web-based functionality.
  • Selenium is an essential tool for achieving efficient and effective web application testing and is an integral part of many quality assurance and software development processes.


Types of Selenium Components:
  • Selenium WebDriver: This is the core component of Selenium. WebDriver provides a programming interface to interact with web browsers such as Chrome, Firefox, Safari, and more. Test scripts are written using WebDriver to automate user interactions with web applications.

  • Selenium IDE (Integrated Development Environment): Selenium IDE is a browser extension (available for Chrome and Firefox) that provides a record and playback mechanism for creating simple test scripts. It's a helpful tool for beginners to get started with Selenium.

  • Selenium Grid: Selenium Grid allows you to distribute test execution across multiple machines and browsers in parallel. It's used for scaling and running tests on different environments and configurations simultaneously.

  • Selenium RC (Remote Control): Selenium RC was an older version of Selenium that has been mostly replaced by Selenium WebDriver. However, some legacy systems may still use Selenium RC. It provides a client-server architecture for executing tests on remote machines.

Write your First Selenium Script:

First Selenium script involves initializing a WebDriver and using it to open a URL. 

        
            import org.openqa.selenium.WebDriver
            import org.openqa.selenium.chrome.ChromeDriver;
            import org.openqa.selenium.chrome.ChromeOptions;
            import io.github.bonigarcia.wdm.WebDriverManager;

            public class MyFirstSeleniumScript {
                public static void main(String[] args) {
                    WebDriverManager.chromedriver().setup();
                    ChromeOptions option = new ChromeOptions();
                    option.addArguments("--remote-allow-origins=*");
                    driver = new ChromeDriver(option);

                    // Open a website in the browser
                    driver.get("https://www.example.com");

                    // Close the browser
                    driver.quit();
                }

    WebDriverManager.chromedriver().setup(); initializes WebDriverManager for ChromeDriver.
    You can replace chromedriver() with firefoxdriver(), edgedriver(), or other WebDriver types as needed.

Locators

Selenium provides several locators to find elements based on their attributes or properties.

The common locators include:

    ID: Locating an element by its unique HTML id attribute. 

    Name: Locating an element by its name attribute. 

    Class Name: Locating an element by its CSS class name. 

    Tag Name: Locating elements by their HTML tag name (e.g., "div," "button"). 

    Link Text: Locating hyperlinks (anchor elements) by the visible text of the link. 

    Partial Link Text: Locating hyperlinks by a partial match of the visible text. 

    XPath: Locating elements using XPath expressions, which provide strong ways to navigate the HTML structure. 

    CSS Selector: Locating elements using CSS selectors.

Example:
WebElement element = driver.findElement(By.id("elementId"));


WebElement:

A WebElement is an interface in Selenium WebDriver that represents an HTML element on a web page. It provides various methods and attributes to interact with the element. When you locate an element using one of the locators, you create a WebElement object that you can use to perform actions like clicking, typing, reading text, and more.

Here are some common actions you can perform with a WebElement:

Click: element.click(): Clicks on the element.

Type Text: element.sendKeys("text"): Types text into input fields.

Get Text: element.getText(): Retrieves the visible text of the element.

Clear Text: element.clear(): Clears text from input fields.

Example:

// Locating an element by ID
WebElement element = driver.findElement(By.id("submitButton"));

// Interacting with the element
element.click(); // Click the button

element.sendKeys("Selenium"); // Type text into an input field

String buttonText = element.getText(); // Get the text on the button

Synchronization and Waits:

Synchronization and waits are essential concepts in Selenium automation testing. They help ensure that your test scripts interact with web elements at the appropriate time, avoiding issues like element not found, element not interactable, or timing-related test failures.

            
                1. Synchronization:

                Synchronization in Selenium is the process of making your test script wait for a certain condition to be met before proceeding with the next steps. It helps handle situations where web elements might not be immediately available or might take some time to load due to various factors like network latency, AJAX requests, or dynamic content loading.

                There are two main types of synchronization in Selenium:

                a. Implicit Wait:

                An implicit wait instructs the WebDriver to wait for a specified amount of time (up to a maximum timeout) when trying to locate an element. If the element is found before the timeout expires, the script proceeds immediately. If the timeout is reached and the element is still not found, a "NoSuchElementException" is raised.

                Example:

                
                    WebDriver driver = new ChromeDriver();
                    driver.manage().timeouts().implicitlyWait(10, TimeUnit.SECONDS);
                

                b. Explicit Wait:

                An explicit wait allows you to wait for a certain condition to be true before proceeding. You can wait for conditions such as element visibility, element clickable, text presence, and more. WebDriverWait is commonly used for explicit waits.

                Example:

                
                    WebDriver driver = new ChromeDriver();
                    WebDriverWait wait = new WebDriverWait(driver, 10); // Maximum timeout of 10 seconds
                    WebElement element = wait.until(ExpectedConditions.visibilityOfElementLocated(By.id("elementId")));
                

                2. Waits:
                It is used to pause the execution of a test script for a specific amount of time.

                There are three types of waits in Selenium:

                a. Sleep:
                The Thread.sleep() method is a wait that pauses the execution for a specified number of milliseconds.

Handling Drop-downs:

The Select class is your tool for dealing with drop-down menus.

Methods to select options:

1. Select By Visible Text 

        Select dropdown = new Select(driver.findElement(By.id("dropdown")));
        dropdown.selectByVisibleText("Option 1");

2. Select By Value 

        Select dropdown = new Select(driver.findElement(By.id("dropdown")));
        dropdown.selectByValue("OptionValue");

3. Select By Index
You can select an option by its index (0, 1..) 

        Select dropdown = new Select(driver.findElement(By.id("dropdown")));
        dropdown.selectByIndex(1);

Handle Windows

In Selenium WebDriver, window handles are used to manage multiple browser windows or tabs when performing web automation testing.
How to work with window handles:

1. Get Current Window Handle:
You can obtain the window handle of the currently active window using the getWindowHandle() method.

String mainWindowHandle = driver.getWindowHandle();

2. Get All Window Handles:
To get all the window handles currently open by your WebDriver instance, use the getWindowHandles() method.

Set < String > allWindowHandles = driver.getWindowHandles();

3. Switch to Specific Window:

driver.switchTo().window(windowHandle);
JavaScript Executor in WebDriver :

The JavaScript Executor in Selenium WebDriver is a built-in feature that allows you to execute JavaScript code directly within your WebDriver automation scripts. This feature is particularly useful when you need to perform actions or validations that are not easily achievable with standard WebDriver commands.

How to use the JavaScript Executor in Selenium WebDriver:

Import the JavascriptExecutor interface.
import org.openqa.selenium.JavascriptExecutor;

// Scroll down the page
JavascriptExecutor jsExecutor = (JavascriptExecutor) driver;
jsExecutor.executeScript("window.scrollBy(0, 500);");


Capturing Screenshot :

Capturing screenshots in Selenium is a useful feature for visual verification and debugging during test automation. You can use the TakesScreenshot interface to capture screenshots.

To capture a screenshot using Selenium WebDriver in Java, you can create a Java program like the following: 


import org.openqa.selenium.By;
import org.openqa.selenium.OutputType;
import org.openqa.selenium.TakesScreenshot;
import org.openqa.selenium.WebDriver;
import org.openqa.selenium.WebElement;
import org.openqa.selenium.chrome.ChromeDriver;
import org.openqa.selenium.io.FileHandler;

import java.io.File;
import java.io.IOException;

public class ScreenshotCaptureExample {
    public static void main(String[] args) {
        // Set the path to the ChromeDriver executable (update as needed)
        System.setProperty("webdriver.chrome.driver", "path/to/chromedriver.exe");

        // Initialize a WebDriver instance for Chrome
        WebDriver driver = new ChromeDriver();

        try {
            // Navigate to a web page (change URL as needed)
            driver.get("https://www.example.com");

            // Locate a WebElement you want to capture (e.g., the logo)
            WebElement logoElement = driver.findElement(By.cssSelector("img.logo"));

            // Capture a screenshot of the WebElement and store it as a file
            File screenshotFile = ((TakesScreenshot) driver).getScreenshotAs(OutputType.FILE);

            // Save the screenshot file to a specified location
            FileHandler.copy(screenshotFile, new File("path/to/save/screenshot.png"));

            System.out.println("Screenshot captured and saved successfully.");
        } catch (IOException e) {
            e.printStackTrace();
        } finally {
            // Close the WebDriver
            driver.quit();
        }
    }
}
